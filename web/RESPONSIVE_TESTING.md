# 📱 響應式設計測試指南
## Responsive Design Testing Guide

本文檔提供完整的響應式設計測試流程，確保網站在所有設備上都能完美顯示。

---

## 🎯 測試目標

- ✅ 所有設備尺寸下的完美顯示
- ✅ 觸控友好的互動體驗
- ✅ 快速載入和流暢動畫
- ✅ 無障礙訪問性
- ✅ 打印友好的排版

---

## 📐 測試斷點 (Breakpoints)

### 手機設備 (Mobile)
- **極小手機**: 320px - 374px (iPhone SE)
- **中等手機**: 375px - 480px (iPhone 12/13/14)
- **大手機**: 481px - 767px (iPhone Pro Max)

### 平板設備 (Tablet)
- **直向平板**: 768px - 834px (iPad Mini)
- **橫向平板**: 835px - 1024px (iPad Air)

### 桌面設備 (Desktop)
- **小桌面**: 1025px - 1199px
- **中桌面**: 1200px - 1399px
- **大桌面**: 1400px - 1919px
- **超寬螢幕**: 1920px+

## 🎨 字體大小檢查

### 基礎字體
- **超小手機** (≤374px): 13px
- **手機** (375-767px): 14px
- **平板** (768-991px): 15px
- **桌面** (≥992px): 16px
- **大桌面** (≥1400px): 18px

### 標題字體
- **H1 (Hero)**: 手機 2.5rem / 桌面 4rem
- **H2 (Section)**: 手機 2rem / 桌面 3rem
- **H3**: 手機 1.5rem / 桌面 2rem

### 可讀性檢查
- [ ] 所有文字清晰可讀
- [ ] 行高足夠 (至少 1.5)
- [ ] 段落間距適當
- [ ] 對比度符合 WCAG AA 標準

## 🐛 常見問題修復

### 問題 1: iOS 輸入框縮放
**解決方案**: 輸入框字體至少 16px
```css
input, select, textarea {
    font-size: 16px;
}
```

### 問題 2: iOS Safari vh 單位問題
**解決方案**: 使用 `-webkit-fill-available`
```css
@supports (-webkit-touch-callout: none) {
    .hero {
        min-height: -webkit-fill-available;
    }
}
```

### 問題 3: 觸控元素太小
**解決方案**: 最小 44x44px
```css
@media (hover: none) and (pointer: coarse) {
    button, .btn {
        min-height: 44px;
        min-width: 44px;
    }
}
```

### 問題 4: 水平滾動
**解決方案**: 檢查固定寬度元素
```css
* {
    max-width: 100%;
    overflow-x: hidden;
}
```

## ✨ 最佳實踐總結

1. **Mobile-First**: 從手機開始設計
2. **Progressive Enhancement**: 逐步增強功能
3. **Touch-Friendly**: 觸控目標至少 44px
4. **Readable Fonts**: 手機字體至少 14px
5. **Flexible Grids**: 使用 flexbox 和 grid
6. **Responsive Images**: 圖片自適應
7. **Performance**: 優化載入速度
8. **Accessibility**: 支援鍵盤和螢幕閱讀器
9. **Testing**: 在真實設備上測試
10. **Iteration**: 持續改進優化
